Student Composite View Meta Data Reports

Target Reports:

Composite ViewDescription
AS_ACADEMIC_HISTORY_EVENT
Contains data pertaining to a student's academic history events.One row per person
AS_ACADEMIC_OUTCOME
Source composite view contains information associated with each academic outcome tracked by the institution. Information is displayed as one row per outcome award, degree or certificate, etc.
AS_ACADEMIC_STANDING
Source composite view contains any academic status values assigned to a student for the term in which they were assigned.
AS_ACTIVE_TERMS
Source composite view contains all students and the terms in which they have activity. This includes academic periods with a new base student record, enrollment activity, academic history, transfer activity, graduation, a cohort entry, sports participation, student attributes, student activities, student work experience, financial aid application or awards, and accounts receivable detail or student contracts.
AS_ADMINISTRATOR
Source composite view contains information about all the persons who will rate admission applicants.
AS_ADMISSIONS_APPLICATION
Source composite view contains information specific to each different application submitted by the applicant.
AS_ADMISSIONS_ATTRIBUTE
Source composite view contains information that identifies special characteristics associated with the applicant for tracking and special reporting.
AS_ADMISSIONS_COHORT
Source composite view contains information that identifies the applicant with a particular group for tracking and reporting purposes.
AS_ADMISSIONS_DECISION
Source composite view contains all of the admissions decisions recorded for each admission application.
AS_ADMISSIONS_RATING
Source composite view contains all administrator ratings recorded for the applicant as a part of the admission evaluation processing.
AS_ADMISSIONS_REQUIREMENT
Source composite view contains all application requirements for each application and whether or not the document or process has been completed for this application.
AS_ADMISSIONS_SOURCE
Source composite view contains information that identifies the reason or source responsible for the applicant submitting the admission application.
AS_ADVISOR
Source composite view contains all current advisors associated with a student.
AS_COURSE_ATTRIBUTE
Source composite view contains all of the course degree attributes associated with the course catalog entry for the academic period.
AS_COURSE_CATALOG
Source composite view contains all of the informational details recorded for the subject or course at a specific time frame or academic period.
AS_COURSE_COREQ
Source composite view contains all of the course catalog entries that are required to be taken in the same academic period as the key subject or course.
AS_COURSE_LEVEL
Source composite view contains all of the levels for which credit can be earned for this subject or course. Included may be undergraduate, graduate, post graduate, etc.
AS_COURSE_OFFERING
Source composite view contains all sections or schedule offerings for a specific academic period.
AS_COURSE_OFFERING_FEES
Source composite view contains all of the fees that will be charged when a student registers for this section. This may include a flat fee for the section or an amount charge per each credit registered for this course. This may also include curriculum values if they were used in defining the rule to assess this course offering fee.
AS_COURSE_PREREQ
Source composite view contains all of the course catalog entries that are required to be taken in a prior academic period as the key subject or course.
AS_COURSE_SCHEDULE
Source composite view contains all of the instructional or delivery methods approved for this course or subject.
AS_COURSE_SUPPLEMENTAL
Source composite view contains additional information about the course that may have the specific purpose to accommodate reporting requirements of the institution.
AS_CURRICULUM
Source composite view contains all active, inactive, current, and and non-current curricula for all students identified in recruiting, admissions, student and academic outcome.
AS_CURRICULUM_FOS
Source composite view contains all active and current curricula for all students identified in recruiting, admissions, and academic outcome as well as all active and current fields of study for Major, Minor, and Concentrations.
AS_ENROLLMENT
Source composite view contains the academic period based fields needed to support the creation of the Student and Academic Study reporting views.
AS_ENROLLMENT_HISTORY
Source composite view contains the academic period based fields needed to support the creation of the Student and Academic Study reporting views.
AS_ENROLLMENT_WITHDRAWAL
Source composite view contains the academic period based fields needed to support the creation of the enrollment withdrawal reporting view.One row per person per academic period per withdrawal record number.
AS_FACULTY
Source composite view contains all the information required to identify a member of academic staff or an instructor at the institution.
AS_FACULTY_ATTRIBUTE
Source composite view contains information that identifies special characteristics associated with the faculty member for tracking and special reporting.
AS_FACULTY_CONTRACT
Source composite view contains all information for an academic period regarding the faculty contracts for that time frame.
AS_FACULTY_DEPARTMENT_COLLEGE
Source composite view contains all of the college and departmental associations for the faculty member in this academic period.
AS_FACULTY_FEEDBACK
Contains informational data about the faculty feedback session rules and the student estimated grade and comment entered by the faculty member of the feedback session. This data may be combined with the faculty feedback detail regarding all issues or recommendations recorded the instructor to obtain complete mandatory and optional input for student registered for the academic period in a student course.One row per academic period per course reference number per feedback session per person
AS_FACULTY_FEEDBACK_DETAIL
Source composite view contains information about faculty feedback recorded for students registered in a student course where feedback by the instructor was required or optional.One row per feedback session per academic period per course reference number per person per feedback (issue or recommendation)
AS_FIELD_OF_STUDY
Source composite view contains all active, inactive, current, and and non-current fields of study for all students identified in recruiting, admissions, student and academic outcome.
AS_GENERAL_STUDENT
Source composite view contains information associated with each program/curriculum for a student. A student may have as many academic study or program records as permitted by the institution ERP system.
AS_GENSTU_END_TERM
AS_GPA_LEVEL
Source composite view contains student's GPA values as stored by type (institutional, transfer or overall) cumulative for all courses by each student level at which the student attended courses and received grades or transfered course work.
AS_GPA_TERM
Source composite view contains student's GPA values as stored for either institutional or transger courses completed by the student and recorded for the academic periods.
AS_INSTRUCTIONAL_ASSIGNMENT
Source composite view contains all the instructors and the teaching assignments they have for the academic period.
AS_LEARNER_CURRICULUM_FOS
Source composite view contains all active and current curricula for all students identified in the Student module as well as all active and current fields of study for Major, Minor, and Concentrations.
AS_MEAL_ASSIGNMENT
Source composite view contains all meal plan information.
AS_MEETING_TIME
Source composite view contains all the meeting time records for each section or schedule offering by academic period.
AS_NONINSTRUCTIONAL_ASSIGNMENT
Source composite view contains all the instructors and all non teaching assignments they have for the academic period. Include may be assignments like administrator, department chair, coach, etc.
AS_OFFERING_COREQ
Source composite view contains all of the co requisite subjects or course for a section or schedule offering that must be taken in the same academic period.
AS_OFFERING_GRADE_MODE
Source composite view contains all of the different grading modes available for the section. Included may be standard, pass/fail, audit or visitor grading, etc.
AS_OFFERING_GRADE_TYPE
Contains grade options or types of grades available for this schedule offering (section). This data will be either a specific grade type specified for the section or one or more values permited for the course identification from the course catalog.
AS_OFFERING_PREREQ
Source composite view contains all of the prerequisite subjects or course for a section or schedule offering that must be completed prior to this academic period.
AS_OUTCOME_HONOR_DEPT
Source composite view contains the outcome departmental honors recorded for a specific academic outcome.
AS_OUTCOME_HONOR_INST
Source composite view contains the outcome institutional honors recorded for a specific academic outcome.
AS_PHONE_ASSIGNMENT
Source composite view contains all telephone assignment information.
AS_PRE_STUDENT
Source composite view contains information on the person who has inquired or applied to be a student or has become a student at the institution. Included is secondary and post secondary information.
AS_RECRUITMENT_ATTRIBUTE
Source composite view contains information that identifies special characteristics associated with the person who has asked for information on the institution. Attributes are to be used for tracking and special reporting.
AS_RECRUITMENT_COHORT
Source composite view contains information that identifies the inquirer with a particular group for tracking and reporting purposes.
AS_RECRUITMENT_INFORMATION
Source composite view contains recruiting information for a person.
AS_RECRUITMENT_LEARNED
Identifies the recruiting learned information for a person.One row per person per academic year per sequence number per learned code.
AS_RECRUITMENT_SOURCE
Source composite view contains the first recruitment source listed for the person.
AS_ROOM_ASSIGNMENT
Source composite view contains room assignments.
AS_SCHEDULE_ATTRIBUTE
Contains attribute information on the course schedule.
AS_SPORT
Source composite view contains all sports associated with a student.
AS_STUDENT_ACTIVITY
Source composite view contains one row for each activity listed for the student.
AS_STUDENT_ATTRIBUTE
Source composite view contains the student attributes associated with the student record.
AS_STUDENT_CLASSIFICATION
AS_STUDENT_COHORT
Source composite view contains all student cohort information.
AS_STUDENT_COURSE_ATTRIBUTE
Source composite view contains course attribute information that indicates which courses may be used to satisfy outcome requirements.
AS_STUDENT_COURSE_ATTR_TRANS
Source composite view contains course attribute information that indicates which courses may be used to satisfy outcome requirements. This may be for institutional or for transfer subjects completed by the student.
AS_STUDENT_COURSE_GRADE_CHG
Source composite view contains information about the grade or grades assigned to the student for this course.
AS_STUDENT_COURSE_HISTORY
Source composite view contains information about the courses taken by a student. It tracks registered, graded and transfer courses.
AS_STUDENT_COURSE_IN_PROGRESS
Source composite view contains information about the courses taken by a student. It tracks registered, graded and transfer courses.
AS_STUDENT_COURSE_REG_AUDIT
Contains audit information for all student registration activity.
AS_STUDENT_COURSE_TRANSFER
Source composite view contains information about the courses taken by a student. It tracks registered, graded and transfer courses.
AS_STUDENT_TRANSFERRED_COURSE
Contains detail data for a each course a student completed at another institution for which this institution is granted equivalent course credit. This reporting view will contain the details as recorded on the received transfer institution transcript as well as the institution equivalent course for which the student is receiving credit. This reporting will show one row for each Transfer Course Detail Repeating Table (SHRTRCE) and Transfer Course Equivalent Repeating Table (SHRTRCE) combination. Therefore they may be more than one row per entry on either the transfere course or the equivalent course. This reporting view should not be used in combination with the STUDENT_COURSE reporting view which will have one row per each Transfer Course Equivalent Repeating Table row.One row per person per academic period per transferred course per equivalent course
AS_STUDENT_WORK_EXPERIENCE
Source composite table contains cooperative education information about the student population. .
AS_TRANSCRIPT_LEVEL_COMMENT
Transcript Comments by Level.One row per person uid per level per comment sequence number.
AS_TRANSCRIPT_TERM_COMMENT
Transcript Comments by Term.One row per person uid per level per academic period per comment sequence number.
AS_VALIDATION
Source composite view contains a number of the source systems' validation table records. This view was meant to be used with reporting tools to create list of values as well as for cleansing purposes in the Enterprise Data Warehouse.
AS_VETERAN_CERTIFICATION
Source composite view contains the academic period data when the institution must report and certify credits for veterans to meet federal government reporting requirements for persons receiving veteran benefits.one row per person per academic period per veteran certification credits
